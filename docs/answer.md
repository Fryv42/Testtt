# Модуль отправки ответов (`static/js/answer.js`)

## Обзор

Модуль реализует клиентскую отправку ответов участника на сервер с валидацией, повторными попытками, обработкой ошибок и подтверждением получения.

Используемый endpoint:

POST /api/v1/sessions/{id}/answer/

---

## Возможности

- Валидация ответа перед отправкой
- Повторные попытки (retry) с экспоненциальной задержкой
- Проверка подтверждения от сервера (`status: "ack"`)
- Блокировка повторной отправки
- Отслеживание статуса (loading / success / error)
- Логирование всех попыток
- Обработка сетевых ошибок

---

## Использование

import AnswerSender from './answer.js';

const sender = new AnswerSender(sessionId);

sender.send(answerId).then(result => {
  if (result.ok) {
    console.log('Ответ успешно отправлен');
  } else {
    console.error('Ошибка отправки:', result.error);
  }
});

---

## API контракт

### Запрос

{
  "answer_id": 1
}

### Успешный ответ

{
  "status": "ack"
}

### Ошибки

- 400 Bad Request — некорректный или отсутствующий `answer_id`
- 404 Not Found — сессия не найдена
- 409 Conflict — дубликат (опционально)
- 500 — ошибка сервера

---

## Класс AnswerSender

### Конструктор

new AnswerSender(sessionId)

Параметры:

- sessionId (number) — идентификатор сессии

---

## Методы

### validate(answerId)

Проверяет корректность ответа.

Возвращает true или false.

---

### send(answerId)

Основной метод отправки.

Поведение:

- Блокирует повторную отправку
- Валидирует данные
- Отправляет запрос с retry
- Обрабатывает ответ сервера

Возвращает:

{ ok: true, data }

или

{ ok: false, error }

---

### _sendWithRetry(payload)

Внутренний метод для повторных попыток.

- Максимум попыток: 3
- Экспоненциальная задержка:

delay = 500ms * номер попытки

---

### _delay(ms)

Вспомогательная функция задержки.

---

### setStatus(status, message)

Обновляет статус отправки.

Возможные статусы:

- loading — отправка
- success — успешно
- error — ошибка

---

### log(payload, status)

Логирует попытки отправки.

Пример:

{
  payload: { answer_id: 1 },
  status: "success",
  timestamp: "2026-04-12T10:00:00Z"
}

---

## Внутреннее состояние

- isSending — блокировка параллельных запросов
- lastPayload — последний отправленный payload
- retryCount — текущая попытка
- maxRetries — максимум попыток (3)

---

## Обработка ошибок

- Некорректный ответ — мгновенная ошибка
- Сетевая ошибка — повтор
- Ошибка сервера — повтор
- Нет ACK — считается ошибкой
- Превышен лимит попыток — возврат ошибки

---

## Примечания

- Дубликаты блокируются на клиенте
- Сервер может дополнительно обрабатывать дубликаты
- Логирование через console.log (можно заменить)
